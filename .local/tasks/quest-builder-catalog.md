# Quest Builder & Catalog Improvements

## What & Why
Доработать форму создания квеста и каталог: добавить подсказки при вводе города, загрузку обложки файлом, кнопку «Поделиться», поле «Кол-во участников», ужесточить валидацию, ограничить количество квестов на странице до 10 и добавить фильтры в UI каталога. Также переименовать термины «Техника безопасности» → «Правила точки» и «Ачивки» → «Достижения» по всему проекту.

## Done looks like
- При вводе города в форме квеста выпадает список подсказок (автокомплит по ранее созданным городам)
- Можно загрузить обложку файлом (JPEG/PNG/WebP ≤ 5 МБ); файл сохраняется через object storage или base64; при превышении размера — понятная ошибка
- Кнопка «Поделиться» на странице квеста копирует ссылку в буфер обмена или открывает Web Share API
- В форме создания квеста есть поле «Максимум участников в команде» (число)
- Нельзя опубликовать квест с менее чем 3 чекпоинтами — показывается ошибка
- Название чекпоинта и текст задания — минимум 20 символов (валидация на клиенте и сервере)
- Каталог показывает 10 квестов на страницу (пагинация изменена с 20 на 10)
- В каталоге работают фильтры: «Длительность» (диапазон минут), «Рядом со мной» (выбрать радиус в км, определяет геолокацию), «Сложность» (три варианта: лёгкий 1–2, средний 3, сложный 4–5)
- Везде в интерфейсе «Техника безопасности» заменена на «Правила точки», «Ачивки» → «Достижения»

## Out of scope
- Хранение файлов на внешнем CDN (достаточно object storage или base64 в БД)
- Изменение логики прохождения квестов

## Steps
1. **Автокомплит города** — На бэкенде добавить эндпоинт, возвращающий список уникальных городов из таблицы квестов. В форме создания квеста подключить выпадающий список, появляющийся при вводе.

2. **Загрузка обложки файлом** — Заменить поле URL на file-input с проверкой типа (image/*) и размера (≤ 5 МБ) на клиенте. На бэкенде добавить эндпоинт загрузки, сохраняющий файл через object storage или как base64 в `cover_url`.

3. **Кнопка «Поделиться»** — Добавить кнопку на страницу детального просмотра квеста. Использовать `navigator.share` (с fallback на копирование в буфер через `navigator.clipboard`).

4. **Поле «Кол-во участников»** — Добавить числовое поле `maxTeamSize` в форму создания квеста, схему Zod и колонку в таблице `quests`.

5. **Ужесточение валидации** — Поднять минимум чекпоинтов до 3 (сервер + UI-счётчик с подсказкой). Добавить валидацию Zod и клиентскую проверку: название чекпоинта ≥ 20 символов, текст задания ≥ 20 символов.

6. **Пагинация 10 на страницу** — Изменить константу `PAGE_SIZE` на 10 в роуте квестов.

7. **Фильтры в UI каталога** — Подключить существующие API-параметры `durationMin`/`durationMax`, `nearLat`/`nearLng`/`radiusKm`, `difficultyMin`/`difficultyMax` к элементам управления в каталоге. Добавить кнопку «Рядом со мной» с запросом геолокации браузера и выбором радиуса.

8. **Переименование терминов** — Глобально заменить «Техника безопасности» → «Правила точки» и «Ачивки» → «Достижения» во всех компонентах, строках и типах.
