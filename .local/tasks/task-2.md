# Gameplay, Scoring & Rules

## What & Why
Улучшить механику прохождения квестов: исправить проверку кодового слова без учёта регистра, зафиксировать время старта и финиша, добавить статус «начато», ввести выбор способа передвижения с бонусными очками, переработать систему начисления очков, запретить проходить квест создателю и повторно одной командой, а рейтинг команд считать только по завершённым квестам.

## Done looks like
- Проверка кодового слова при прохождении чекпоинта нечувствительна к регистру
- У игровой сессии фиксируется `startedAt` (момент старта) и `finishedAt` (момент финиша/отказа)
- Статусы сессии: `started` → `in_progress` → `completed` / `abandoned`
- Перед началом квеста игрок/команда выбирает способ передвижения: пешком / на транспорте / на общественном транспорте / по грунтовкам / без дорог — от этого зависит множитель бонусных очков
- Очки начисляются: за каждый пройденный чекпоинт + бонус за завершение квеста + бонус за скорость (чем быстрее — тем больше) + множитель сложности квеста + множитель способа передвижения
- Один квест засчитывается команде/игроку только один раз (повторная попытка заблокирована с пояснением)
- Нельзя начать квест, если в команде есть его создатель (блокировка с сообщением)
- В рейтинге команд учитываются только очки за **завершённые** квесты

## Out of scope
- Изменение интерфейса достижений
- Реальный GPS-трекинг маршрута

## Steps
1. **Нечувствительность к регистру** — В роуте проверки ответа чекпоинта привести введённое слово и эталонный ответ к нижнему регистру перед сравнением.

2. **Время старта и финиша** — Добавить колонки `started_at` и `finished_at` в таблицу `play_sessions`. Заполнять `started_at` при создании сессии, `finished_at` — при статусе `completed` или `abandoned`.

3. **Статус `started`** — Добавить начальный статус `started` в enum сессий. Сессия создаётся со статусом `started`, переходит в `in_progress` при достижении первого чекпоинта.

4. **Выбор способа передвижения** — Добавить поле `travel_mode` (enum: `foot` / `transport` / `public_transport` / `dirt_road` / `off_road`) в таблицу `play_sessions`. В UI добавить шаг выбора способа передвижения перед стартом квеста с описанием бонуса для каждого варианта.

5. **Система начисления очков** — Реализовать функцию подсчёта очков: базовые очки за чекпоинт + бонус завершения + бонус скорости (процент от максимального времени квеста) + множитель сложности квеста (difficulty × коэффициент) + множитель `travel_mode`. Начислять очки пользователю и команде при завершении.

6. **Ограничение «один раз»** — При попытке начать квест проверять, есть ли у команды/игрока завершённая сессия по этому квесту. Если есть — вернуть 403 с понятным сообщением.

7. **Запрет создателю** — При старте командного квеста проверять, является ли кто-то из участников команды автором квеста. Если да — вернуть 403.

8. **Рейтинг команд** — Изменить запрос лидерборда команд: суммировать очки только из сессий со статусом `completed`.
