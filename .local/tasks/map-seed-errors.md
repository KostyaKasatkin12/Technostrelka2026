# Map Clustering, Seed Data & Error Pages

## What & Why
Добавить кластеризацию маркеров на карте Яндекса при отдалении, исправить координаты и маршруты существующих квестов (некоторые чекпоинты образуют V-образные петли и несвязные маршруты), создать демо-аккаунт модератора для жюри, наполнить БД реалистичными тестовыми данными (квесты, пользователи, команды, прохождения) и добавить страницы обработки HTTP-ошибок 400/401/403/404.

## Done looks like
- При отдалении на карте маркеры квестов/чекпоинтов объединяются в кластеры с числом; при приближении — раскрываются
- Существует аккаунт с логином `moderator` и паролем `demo123` с ролью модератора (создаётся через seed-скрипт, не хардкодится в коде)
- В базе: 8–12 квестов в одном городе (разные районы), каждый квест содержит 3–7 чекпоинтов; 6–10 пользователей; 3–5 команд; 10–20 записей о прохождениях (статус `completed`) — все данные реалистичные, на русском языке
- При переходе на несуществующую страницу (404) отображается страница с кнопкой «На главную»
- При ошибках 401 (не авторизован) и 403 (нет доступа) — соответствующие страницы с понятным описанием
- Все API-роуты возвращают структурированный JSON с кодом ошибки и сообщением для 400/401/403/404/500

## Out of scope
- Изменение логики авторизации
- Реальные фотографии мест (достаточно placeholder-изображений)

## Steps
1. **Кластеризация маркеров** — Подключить встроенный Clusterer Яндекс Карт (ymaps.Clusterer или аналог из используемого SDK) к карте с квестами и чекпоинтами. Настроить стили кластеров в фирменных цветах приложения.

2. **Страницы ошибок** — Создать React-компоненты для страниц 401, 403, 404. Подключить их через роутер (`wouter`): несуществующие пути → 404, перенаправление при ошибках API → соответствующие страницы. На бэкенде убедиться, что все роуты возвращают JSON `{ error, message }` для всех статусов ошибок.

3. **Seed-скрипт** — Написать или дополнить существующий seed-скрипт: создать пользователя `moderator`/`demo123` (роль `moderator`), 6–10 обычных пользователей, 3–5 команд, 8–12 квестов (один город, разные районы, 3–7 чекпоинтов каждый, статус `published`), 10–20 завершённых сессий прохождения с реалистичными данными очков и временем. Скрипт должен быть идемпотентным (повторный запуск не дублирует данные).
